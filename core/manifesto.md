# REF Manifesto

Requirements engineering exists to reduce ambiguity.

Procurement exists to identify the supplier providing the best overall value.

Good requirements reduce uncertainty for suppliers.

Good requirements reduce effort for evaluators.

Good requirements improve procurement outcomes.

Every unnecessary word increases interpretation risk.

Every subjective expression increases evaluation risk.

Every combined obligation increases implementation risk.

Every scoreable requirement increases evaluation effort.

Therefore:

- Use the simplest evaluation model that preserves procurement value.
- Split requirements whenever obligations can be evaluated independently.
- Reserve scored evaluation for capabilities that objectively differentiate suppliers.
- Separate requirements from evidence.
- Separate requirements from explanations.
- Use configuration instead of assumptions.
- Store engineering knowledge, not customer-specific content.
- Generate requirements from capabilities, not from previous tenders.

## Golden rule

A requirement shall exist because a capability requires it, not because it appeared in a previous tender.
