# ADR 0001: Capability-first requirements engineering

**Status:** Accepted  
**Date:** 2026-07-08

## Context

Initial discussion focused on rewriting existing requirements. This risks preserving the structure and omissions of previous tenders.

## Decision

REF shall engineer requirements from capabilities rather than only editing existing requirement text.

## Consequences

- Existing requirements are mapped to capabilities.
- Missing capabilities can be detected.
- Requirements can be generated from project configuration and knowledge packs.
- Previous tenders are not treated as authoritative baselines.

## Rationale

A requirement shall exist because a capability requires it, not because it appeared in a previous tender.
