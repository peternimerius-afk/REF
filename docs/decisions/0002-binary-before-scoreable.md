# ADR 0002: Binary before scoreable

**Status:** Accepted  
**Date:** 2026-07-08

## Context

Public procurement evaluation effort increases with every scoreable requirement.

## Decision

REF shall prefer binary SHALL requirements by default. Scoreable requirements shall only be used where measurable supplier differentiation creates procurement value.

## Consequences

- Split obligations before scoring.
- Mandatory baseline compliance is binary.
- Maturity may be scoreable where justified.
- Maximum scoreable ratio is configurable by policy.

## Rationale

Use the simplest evaluation model that preserves procurement value.
