# Knowledge Packs

**Version:** 0.1.0  
**Status:** Draft

Knowledge packs provide reusable engineering knowledge for capabilities, domains, technologies, regulations, and solution types.

Knowledge packs shall not contain customer-specific tender content.

## Purpose

Knowledge packs help REF answer:

- Which capabilities are expected?
- Which requirements are usually mandatory?
- Which aspects may be scoreable?
- What evidence is normally useful?
- What common mistakes should be avoided?

## Structure

A capability package should use this structure:

```text
knowledge/<group>/<capability>/
  capability.yaml
  engineering.md
  evidence.md
  requirements.md
  scoring.md
  examples.md
```

## Content boundary

Knowledge packs may contain:

- Generic engineering guidance
- Capability metadata
- Typical evidence
- Common mistakes
- Generic examples
- Scoring guidance

Knowledge packs shall not contain:

- Customer names
- Supplier names
- Project-specific requirements
- Tender decisions
- Pricing
- Confidential documents
- Non-anonymized regression material
