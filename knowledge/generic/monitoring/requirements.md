# Monitoring Requirement Patterns

## Mandatory baseline

> The solution shall monitor service availability and critical integrations.

Evaluation:

> Yes / No

Evidence:

> Monitoring documentation

## When to split

Split this capability when the requirement also includes:

- Evidence requests
- Reporting obligations
- Testing obligations
- Support obligations
- Separate technical mechanisms
- Contractual remedies

## Scoreable candidate

Scoreable variant: `observability_maturity`

Use only if higher maturity creates measurable procurement value.
