# Authentication Requirement Patterns

## Mandatory baseline

> The solution shall support federated authentication.

Evaluation:

> Yes / No

Evidence:

> Identity integration guide

## When to split

Split this capability when the requirement also includes:

- Evidence requests
- Reporting obligations
- Testing obligations
- Support obligations
- Separate technical mechanisms
- Contractual remedies

## Scoreable candidate

Scoreable variant: `identity_maturity`

Use only if higher maturity creates measurable procurement value.
