# Encryption Requirement Patterns

## Mandatory baseline

> Data shall be encrypted in transit and at rest.

Evaluation:

> Yes / No

Evidence:

> Security architecture

## When to split

Split this capability when the requirement also includes:

- Evidence requests
- Reporting obligations
- Testing obligations
- Support obligations
- Separate technical mechanisms
- Contractual remedies

## Scoreable candidate

Scoreable variant: `security_maturity`

Use only if higher maturity creates measurable procurement value.
