# Audit Logging Requirement Patterns

## Mandatory baseline

> Security-relevant events shall be audit logged.

Evaluation:

> Yes / No

Evidence:

> Logging documentation

## When to split

Split this capability when the requirement also includes:

- Evidence requests
- Reporting obligations
- Testing obligations
- Support obligations
- Separate technical mechanisms
- Contractual remedies

## Scoreable candidate

Scoreable variant: `logging_maturity`

Use only if higher maturity creates measurable procurement value.
