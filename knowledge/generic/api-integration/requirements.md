# API Integration Requirement Patterns

## Mandatory baseline

> The solution shall provide documented APIs for integration.

Evaluation:

> Yes / No

Evidence:

> API documentation

## When to split

Split this capability when the requirement also includes:

- Evidence requests
- Reporting obligations
- Testing obligations
- Support obligations
- Separate technical mechanisms
- Contractual remedies

## Scoreable candidate

Scoreable variant: `api_maturity`

Use only if higher maturity creates measurable procurement value.
