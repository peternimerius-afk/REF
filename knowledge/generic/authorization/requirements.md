# Authorization Requirement Patterns

## Mandatory baseline

> The solution shall support role-based access control.

Evaluation:

> Yes / No

Evidence:

> Role model

## When to split

Split this capability when the requirement also includes:

- Evidence requests
- Reporting obligations
- Testing obligations
- Support obligations
- Separate technical mechanisms
- Contractual remedies

## Scoreable candidate

Scoreable variant: `access_control_maturity`

Use only if higher maturity creates measurable procurement value.
