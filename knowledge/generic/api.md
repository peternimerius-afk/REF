# API Capability

## Purpose

Ensure that the solution can integrate with external systems through documented interfaces.

## Typical mandatory requirements

- The solution shall provide documented APIs.
- API authentication shall be supported.
- API versioning shall be documented.

## Typical scoreable differentiator

API maturity may be scoreable when integration capability is central to procurement success.

Possible maturity dimensions:

- OpenAPI specification;
- sandbox environment;
- webhook support;
- SDKs;
- developer portal;
- versioning policy;
- rate limiting and monitoring.

## Common mistakes

- Combining API support, documentation, webhooks and sandbox into one requirement.
- Scoring basic API existence instead of API maturity.
- Mixing evidence requests into the requirement.
